# OpenAI API Labs

This directory contains a curated set of educational labs for learning the
OpenAI API with Python. The goal of the collection is to stay practical,
readable, and progressively structured: each script focuses on one main idea,
while the notebooks provide more exploratory and reference-oriented material.

The labs are intentionally hands-on. They cover common API patterns, structured
outputs, tool usage, multimodal workflows, embeddings, media generation, cost
awareness, and model comparison. The repository also keeps a few generated
audio, image, and video assets so the project can show concrete examples of
what OpenAI models can produce.

## What This Directory Contains

- `scripts/`: executable Python labs, organized as numbered scenarios
- `notebooks/`: Jupyter notebooks for quick reference and deeper exploration
- `scripts/audio_outputs/`: sample audio files generated by the labs
- `scripts/images_outputs/`: sample images generated by the labs
- `scripts/video_outputs/`: sample videos and preview assets generated by the labs


## Prerequisites

Before running the OpenAI labs, make sure you have:

- Python `3.13.12` or newer
- an OpenAI API key
- a single `.env` file at the repository root

Example root `.env` file:

```env
OPENAI_API_KEY=your_openai_api_key_here
```

Some optional labs also look for additional variables:

- `OPENAI_SAMPLE_IMAGE_URL`
- `OPENAI_SAMPLE_IMAGE_PATH`
- `OPENAI_SAMPLE_AUDIO_PATH`
- `OPENAI_SAMPLE_FOREIGN_AUDIO_PATH`
- `OPENAI_PROMPT_ID_COMPOSER`
- `OPENAI_PROMPT_ID_FILE_SUMMARY`

## Python Dependencies

This directory contains its own `pyproject.toml` so users can install only the
dependencies needed for the OpenAI labs instead of installing the entire
repository stack.

Base dependencies required for the scripts:

- `openai`
- `python-dotenv`
- `pydantic`
- `httpx`

Optional notebook dependencies:

- `jupyterlab`
- `ipykernel`

## Installation

### Option 1: Install With `uv` (Recommended)

If you want only the Python scripts:

```bash
cd 01_openai
uv sync
```

If you also want to run the Jupyter notebooks:

```bash
cd 01_openai
uv sync --extra notebooks
```

You can then run a script from the repository root:

```bash
uv run --project 01_openai python 01_openai/scripts/01_basic_openai_calls.py
```

Or from inside `01_openai`:

```bash
uv run python scripts/01_basic_openai_calls.py
```

To launch JupyterLab:

```bash
cd 01_openai
uv run --extra notebooks jupyter lab
```

### Option 2: Install Without `uv`

Create and activate a virtual environment first:

```bash
cd 01_openai
python3.13 -m venv .venv
source .venv/bin/activate
```

Install the dependencies for the scripts:

```bash
pip install .
```

Install the notebook dependencies as well:

```bash
pip install ".[notebooks]"
```

Then run a script:

```bash
python scripts/01_basic_openai_calls.py
```

And launch JupyterLab when needed:

```bash
jupyter lab
```

## Running The Labs

### Running The Python Scripts

Each numbered file in `scripts/` is a standalone lab. Run them one by one,
for example:

```bash
uv run --project 01_openai python 01_openai/scripts/02_structured_output_scenarios.py
```

or, if you are already inside `01_openai`:

```bash
uv run python scripts/02_structured_output_scenarios.py
```

### Running The Jupyter Notebooks

The notebooks live in `notebooks/` and complement the scripts with a more
interactive learning experience.

Start JupyterLab from inside `01_openai`:

```bash
uv run --extra notebooks jupyter lab
```

Then open:

- `notebooks/openai_quick_reference.ipynb`
- `notebooks/structured_output_deep_dive.ipynb`
- `notebooks/provider_comparison_openai.ipynb`

### Which Installation Path Should You Choose?

Use the base installation if:

- you only want to run the numbered `.py` labs
- you do not need Jupyter

Install the notebook extra if:

- you want to open and execute the notebooks
- you want a more exploratory workflow

This setup keeps the OpenAI labs modular. A user who only wants `01_openai`
does not need to install dependencies for Anthropic, Google, or other folders
in the repository.

## Labs Overview

### Core Request Patterns

- `01_basic_openai_calls.py`
  Covers simple text generation, instruction-style prompting, rewriting, and
  JSON-shaped outputs requested through prompting alone.

- `05_streaming_scenarios.py`
  Shows how to stream text incrementally, inspect streaming event types, and
  stream function-call arguments before local execution.

- `08_error_handling_scenarios.py`
  Covers common failure modes such as missing keys, timeouts, rate limits,
  schema validation failures, refusals, and local input validation.

### Structured Outputs and Typed Responses

- `02_structured_output_scenarios.py`
  Introduces typed parsing with Pydantic, nested extraction, and strict JSON
  schema outputs.

- `16_recursive_structured_output_scenarios.py`
  Demonstrates recursive structured output generation through a nested UI
  component tree.

- `17_moderation_structured_output_scenarios.py`
  Uses structured outputs for moderation-style classification, with a compact
  decision object suitable for programmatic inspection.

### Tool Use and Function Calling

- `03_function_calling_scenarios.py`
  Covers application-defined function calling, including single calls,
  multi-step calls, and required tool usage.

- `04_tool_calling_scenarios.py`
  Focuses on OpenAI-hosted built-in tools such as web search and code
  interpreter.

- `15_reusable_prompt_scenarios.py`
  Demonstrates hosted prompt templates with variables, including a scenario
  that combines reusable prompts with uploaded file input.

### Usage, Tokens, and Cost Awareness

- `06_token_cost_tracking.py`
  Explains how to read usage metadata, estimate request cost, pre-count input
  tokens, and compare reasoning effort settings.

- `07_conversation_state_and_costs.py`
  Shows how conversation state changes token usage and cost, comparing
  stateless message history with `previous_response_id` workflows.

- `14_model_comparison_scenarios.py`
  Benchmarks several OpenAI models on the same task, comparing answer quality,
  latency, estimated cost, and JSON reliability.

### Multimodal and Embedding Workflows

- `09_multimodal_inputs.py`
  Covers text plus image URL, text plus local image as a data URL, and text
  plus local PDF input.

- `10_embeddings_scenarios.py`
  Demonstrates semantic similarity, a tiny semantic search workflow, and
  near-duplicate detection with cosine similarity.

### Media Generation

- `11_image_generation_scenarios.py`
  Covers image generation with the Images API, including multiple outputs and a
  higher-quality generation example with rough cost estimation.

- `12_audio_scenarios.py`
  Covers text-to-speech, voice variation, speech transcription, and audio
  translation to English.

- `13_video_scenarios.py`
  Demonstrates short video generation, budget-aware scenario design, asset
  download, and preview metadata retrieval.

## Notebooks Overview

- `openai_quick_reference.ipynb`
  A compact notebook for everyday OpenAI SDK usage. It acts as a quick-start
  reference for plain text generation, structured outputs, streaming,
  embeddings, and richer Responses API inputs.

- `structured_output_deep_dive.ipynb`
  A more detailed notebook focused on structured outputs. It expands the
  scripts by showing typed parsing, nested extraction, strict schemas,
  recursive output structures, and moderation-style classification in a more
  exploratory format.

- `provider_comparison_openai.ipynb`
  A notebook version of the model-comparison mindset. It uses OpenAI models as
  a consistent baseline to compare cost, latency, output quality, and JSON
  stability with an interactive workflow.

## Main OpenAI Use Cases Covered

Taken together, the scripts and notebooks cover the following major OpenAI API
use cases:

- basic text generation
- instruction following
- rewriting and content transformation
- structured output parsing with Pydantic
- strict JSON schema responses
- recursive structured output generation
- moderation-style classification
- custom function calling
- hosted built-in tools
- reusable hosted prompts
- streaming responses
- token counting and cost estimation
- stateful multi-turn conversations
- error handling and refusal inspection
- multimodal inputs with images and PDFs
- embeddings and semantic similarity
- image generation
- audio generation, transcription, and translation
- video generation
- model comparison and trade-off evaluation

## Reusable Helper Module

The file `scripts/openai_client.py` contains a small wrapper around recurring
OpenAI SDK patterns. It is intentionally lightweight and educational rather
than abstract:

- client creation
- text generation
- structured output parsing
- text streaming
- embedding generation

It is particularly useful in notebooks and quick experiments when you want to
reduce boilerplate without hiding the underlying API shape.

## Philosophy Of The Labs

These labs are designed to stay:

- simple enough to read in one sitting
- concrete enough to run and modify immediately
- modular enough to isolate one idea per file
- practical enough to serve as real future references

The collection is not meant to be a framework. It is meant to be a clear,
useful learning workspace for experimenting with the OpenAI API.
